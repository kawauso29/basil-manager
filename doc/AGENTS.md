# ドキュメント作成ガイド

このディレクトリには、basil-manager の設計と仕様を記録します。
実装を変更する際は、関連するドキュメントも同じ変更に含めてください。

## ファイル構成

- `er.md`: 業務テーブル全体の関連を示すER図
- `db/<table>.md`: 業務テーブルごとの詳細仕様
- `enum/README.md`: Rails enumとenum_helpの共通運用ルール

`db` 配下のファイル名は、Railsのモデル名に合わせた単数形の
snake_caseとします。たとえば、`plants` テーブルの仕様は
`db/plant.md` に記載します。

Solid Cache、Solid Queue、Solid Cableなど、ライブラリが管理する
内部テーブルは、アプリケーション固有の判断が必要になった場合を除き、
ここで扱う業務テーブル仕様の対象外とします。

## テーブル仕様に記載する内容

`db/<table>.md` には、次の項目を記載します。

1. テーブルの目的
2. カラム名、型、NULL許可、デフォルト値、キー、説明
3. インデックスと一意制約
4. 他テーブルとの関連
5. 値の制約や更新条件などの業務ルール
6. 未確定事項

## 更新ルール

- migrationやモデルの変更時は、対象テーブルの仕様書を更新する
- テーブルや関連を追加・削除した場合は、`er.md` も更新する
- ER図、テーブル仕様、実装の名称と関連を一致させる
- 実装や合意から確認できない仕様を推測で確定しない
- 未確定の内容は「未確定事項」に明記する
